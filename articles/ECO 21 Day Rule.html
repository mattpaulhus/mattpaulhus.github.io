The Engineering Change Order 21-Day Rule

Keeping the Engineering Change Process Moving

April 2021 Matt Paulhus

Business Problem

How can we set up an automated business rule so that documentation and product changes that linger in review status are returned to the originator if not actioned?

Method

I programmed the Agile Product Lifecycle Management system event scripting, a Java-based scripting application, to automate the notification and eventual workflow state change.

As part of this project, I needed to write the program that would do the following:

  • Search for Engineering Change Orders (ECOs)(and a few of our other workflows) that had been in Review status for 21 days.
  • Determine if the ECO was still "active" based on the number of recent workflow comments and newly added reviewers.
  • If the ECO met the criteria, and it was at 14 days, notify the Originator and remaining reviewers.
  • Return to Pending status ECOs in Review status for 21 days that did not meet the criteria, and notify the Originator.

Outcome

Such a great success. Our ECO cycle time never goes above 21 days because reviewers understand the limited time window they have to review, comment, accept and/or reject the change. This also makes sure ECOs remain applicable to the current environment.
