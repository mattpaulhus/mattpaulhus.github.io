Notifying the Last Reviewer

Keeping the Engineering Change Process Moving

January 2018 Matt Paulhus

Business Problem

How can we keep the documentation and product changes moving by notifying the last reviewer?

Method

I programmed the Agile Product Lifecycle Management system's Java-based scripting handler to automate a notification and send an email to the individual if he or she is the last person to review an Engineering Change Order in Review status.

As part of this project, I needed to write the program that would do the following:

  • Each time a reviewer approver while in Review status, count the number of remaining reviewers.
  • If there is one more approver at the current status, notify the originator and last approver.

Outcome

Currently working on the data analysis to get more statistics. Nonetheless, I have also had positive customer feedback so far as many find it helpful.
